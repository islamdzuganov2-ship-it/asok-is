### T-22. Карточка «Качество по подхарактеристикам во времени» — сворачиваемая **[фронт]**
- Карточку сделать разворачиваемой/сворачиваемой (`CollapsibleCard`), тумблер «в самом верху» карточки.
- Логика раскрытия по фильтру характеристики:
  - выбрано «Все характеристики» **или** конкретная характеристика → карточка **раскрывается**
    и показывает соответствующие подхарактеристики (по выбору фильтра);
  - кнопка **«Скрыть все»** / «Показать все» сворачивает и разворачивает карточку целиком.
- Заголовок отражает контекст: `Качество по подхарактеристикам во времени (N)` с тегом выбранной

### T-24. Второе представление — пузырьковая диаграмма (bubble timeline) **[фронт]**
- Новая карточка «Пузырьковая карта задач» рядом с существующей «читаемой» диаграммой Ганта.
- Ось X — **время**: от самого раннего старта поставленных задач до самой поздней даты
  исполнения (срока) из плана. Ось Y — swimlane по системам (или по характеристикам при
  выбранной одной ИС).
- Пузырьки **светятся** тремя цветами (glow = `shadowBlur`/`shadowColor`):
  - **красный** — просрочено (срок < сегодня, не выполнено);
  - **жёлтый** — зона риска (до срока ≤ порога, по умолчанию 14 дней; не выполнено);
  - **зелёный** — в плане (срок с запасом или задача выполнена).
- Клик по пузырьку → открывает карточку задачи (та же модалка, что у диаграммы Ганта).
- Под диаграммой — **сворачиваемый список задач** в виде таблицы:
  колонки **«Тема задачи» · «Ответственный» · «Срок исполнения»** (+ статус-тег, ИС).
  Клик по строке таблицы → открывает ту же карточку задачи.
- Легенда трёх цветов зон под диаграммой.

## 4. Дашборд МК → «Основное» (`ManagerDashboard.tsx`)

### T-26. Переименование дашборда в «Основное» **[фронт]**
- Заголовок страницы и пункт меню: «Основное» (вместо «Менеджер по качеству»).

### T-27. Карточка-пирог «Профиль качества по характеристикам» **[фронт]**
- Добавить карточку с **круговой диаграммой (пирог/донат)** с делением на 8 характеристик ISO/IEC 25010
  выбранной ИС; сектор окрашен по RAG-уровню, значение — балл характеристики.
- Диаграмма — **интерактивный селектор**: клик по сектору выбирает характеристику и запускает
  каскад раскрытия (T-28/T-29). Центр доната — интегральный балл ИС.
- **Дублёр-селектор (доступность):** под пирогом — компактная строка «фишек» характеристик
  (клик/Enter выбирает ту же характеристику). Гарантирует выбор с клавиатуры и при недоступности
  canvas; визуально подсвечивает активную характеристику по RAG.

### T-28. Карточка метрик появляется только по выбранной характеристике + «спрятать» **[фронт]**
- Карточка «Метрики характеристики «X»» (подхарактеристики) **не показывается по умолчанию**;
  появляется и раскрывается **только когда выбрана характеристика** (клик по сектору пирога /
  по списку характеристик).
- На карточке — опция **«Спрятать»** (свернуть/скрыть выбор характеристики).

### T-29. Каскадное раскрытие карточек по выбранной характеристике **[фронт]**
Порядок раскрытия при выборе характеристики:
1. Раскрывается карточка **подхарактеристик** (метрики характеристики).
2. Если на характеристику **есть завязанные меры/намерения** — раскрывается карточка
   «Меры и намерения» (с этой информацией).
3. Если мер на уровне характеристики **нет** — карточки «Меры и намерения» и
   «Профессиональные суждения» **не раскрываются**, пока пользователь не выберет
   подхарактеристику, у которой есть меры/суждения. Тогда раскрываются соответствующие карточки.

### T-30. Переименование карточки суждений **[фронт]**

## 5а. Доработки v15.1 (по замечаниям) — **реализовано, проверено в браузере**

### T-32. «План задач» — пузырьки по ОТВЕТСТВЕННЫМ, кластеры-пироги с fan-out **[фронт]**
- Ось Y пузырьковой карты — **ответственные** (не системы). Системы в этом виде **не упоминаются**:
  убраны фильтр по системе, «ИС» из подписей Ганта, из списка задач и из карточки задачи.
- Задачи одного ответственного с **совпадающей датой исполнения** собираются в «пирог» (донат
  с сегментами по зонам + счётчик); при **наведении** пирог разъезжается на отдельные пузырьки —
  можно выбрать нужную. Клик по пузырьку/строке списка открывает задачу.
- Реализация: `frontend/src/components/TaskBubbleTimeline.tsx` (HTML/CSS, не canvas — плавный
  fan-out и доступность), `TaskPlanDashboard.tsx` (ECharts-версия удалена).

### T-33. «Основное» — «Метрики» под «Профиль качества», вертикальный стек **[фронт]**
- Порядок: «Профиль качества» (полноширинная, пирог + «фишки» рядом) → «Метрики характеристики»
  под ней → далее остальные карточки.

### T-34. «Основное» — «Выработка мер»/«Профессиональные суждения» строго по данным **[фронт]**
- «Выработка мер» завязана на выбранную характеристику и раскрывается только при систематической
  зоне по ней (`MeasureDevelopmentPanel`: props `characteristic` + `hideWhenEmpty` → self-hide).
- «Профессиональные суждения» раскрываются только при наличии суждений по характеристике/
  подхарактеристике (`FilledJudgmentsCard`: `hideWhenEmpty`).
- «Меры и намерения» — только при наличии мер (без изменений).

### T-35. Управленческий дашборд — «Реестр мер качества» скрыт по умолчанию **[фронт]**
- Карточка реестра **скрыта по умолчанию**, раскрывается/сворачивается кнопкой. `ExecutiveDashboard.tsx`.

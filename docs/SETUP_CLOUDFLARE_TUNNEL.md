# Полная пошаговая инструкция — публикация asokis.ai через Cloudflare Tunnel (named)

Результат: стабильный адрес `https://asok.asokis.ai`, который ведёт на ваш ПК через
зашифрованный туннель Cloudflare. **Не нужны** проброс портов, белый IP и открытый firewall.
Туннель сам устанавливает исходящее соединение наружу.

Ваши данные (подставлены заранее):
- Домен: `asokis.ai`
- Nameservers Cloudflare: `desi.ns.cloudflare.com`, `patrick.ns.cloudflare.com`
- Рекомендуемый адрес приложения: `asok.asokis.ai`

---

## 0. Предусловия (один раз)
1. Установлен и запущен **Docker Desktop** (Windows). Проверка в PowerShell:
   ```powershell
   docker version
   docker compose version
   ```
2. Проект склонирован, вы в его папке `C:\Users\adiga\projects\asok-is`.
3. Базовый стенд поднимается локально:
   ```powershell
   docker compose up -d
   ```
   Открыть `http://localhost:3000` — приложение работает.

---

## 1. Сделать домен активным в Cloudflare
> Если в панели Cloudflare домен `asokis.ai` уже в статусе **Active** — пропустите шаг.

1. Зайдите к вашему **регистратору** домена (где покупали `asokis.ai`).
2. Найдите раздел **Nameservers (DNS-серверы)**.
3. Удалите текущие NS и впишите ровно два от Cloudflare:
   - `desi.ns.cloudflare.com`
   - `patrick.ns.cloudflare.com`
4. Сохраните. Отключите **DNSSEC** у регистратора, если он включён (можно включить позже в Cloudflare).
5. Ждите смены статуса на **Active** в Cloudflare (обычно 1–2 часа, до 24 ч).
   Проверка статуса: панель Cloudflare → домен `asokis.ai` → вверху статус.

> Туннель технически поднимется и до Active, но публичный хостинг `asok.asokis.ai`
> заработает только после активации зоны.

---

## 2. Открыть Cloudflare Zero Trust (бесплатно)
1. Войдите на `https://one.dash.cloudflare.com` (тот же аккаунт, что и домен).
2. Если входите впервые — Cloudflare попросит **название команды (team name)**: придумайте любое
   (напр. `asokis`), выберите план **Free**, привязка карты для Free не списывает деньги.

---

## 3. Создать туннель
1. Слева: **Networks → Tunnels** (в старых интерфейсах: **Access → Tunnels**).
2. Кнопка **Create a tunnel**.
3. Тип коннектора: **Cloudflared** → **Next**.
4. Имя туннеля: `asok` → **Save tunnel**.

---

## 4. Скопировать токен туннеля (НЕ запускать их команду)
На экране установки Cloudflare покажет команды для разных ОС с длинным токеном вида
`eyJhIjoi... (очень длинная строка)`.
1. Не выполняйте предлагаемую команду установки — мы запускаем cloudflared в Docker.
2. Скопируйте **только токен** — это строка после `--token` (начинается на `eyJ`).
   Можно скопировать из любой вкладки (Windows/Linux) — токен один и тот же.
3. Нажмите **Next** (перейдём к публичному хостнейму).

---

## 5. Прописать публичный хостнейм (какой адрес → какой сервис)
1. На шаге **Route tunnel** → вкладка **Public Hostnames** → **Add a public hostname**.
2. Заполните:
   - **Subdomain**: `asok`
   - **Domain**: `asokis.ai`
   - **Path**: оставить пустым
   - **Type**: `HTTP`
   - **URL**: `frontend:3000`
3. (Опционально) Разверните **Additional application settings → TLS** — ничего менять не нужно,
   так как до фронтенда идёт обычный HTTP внутри docker-сети.
4. **Save**. 

> Хотите адрес на корне (`https://asokis.ai`) вместо поддомена — на этом шаге выберите
> Domain `asokis.ai` и оставьте Subdomain пустым. Бесплатный Universal SSL покрывает и корень,
> и поддомены первого уровня.

---

## 6. Вставить токен в проект
1. Откройте файл `.env` в корне проекта (НЕ `.env.example`). Если его нет — создайте, скопировав
   из `.env.example`.
2. Добавьте/замените строку (вставьте свой скопированный токен целиком):
   ```
   TUNNEL_TOKEN=eyJhIjoi... (ваш длинный токен)
   ```
3. Сохраните файл. Токен — секрет: не публикуйте его и не коммитьте `.env` в git.

---

## 7. Запустить туннель
```powershell
docker compose --profile named up -d
docker compose logs -f tunnel-named
```
В логах дождитесь строк:
```
INF Registered tunnel connection connIndex=0 ...
INF Registered tunnel connection connIndex=1 ...
```
Это значит, что коннектор подключился к Cloudflare. (Ctrl+C — выйти из логов, контейнер
продолжит работать.)

---

## 8. Проверка
1. Откройте `https://asok.asokis.ai` в браузере — должно загрузиться приложение.
2. Войдите демо-учёткой (если включён DEMO_MODE): `admin` / `Admin123!`.
3. Если «зависает» 10–30 секунд после первого старта — это нормально (DNS-прогрев Cloudflare).

---

## 9. Управление
- Остановить только публичный доступ (локальный `:3000` останется):
  ```powershell
  docker compose stop tunnel-named
  ```
- Снова включить:
  ```powershell
  docker compose --profile named up -d
  ```
- Полностью убрать контейнер туннеля:
  ```powershell
  docker compose rm -sf tunnel-named
  ```

---

## 10. Диагностика проблем
| Симптом | Причина / решение |
|--------|-------------------|
| `https://asok.asokis.ai` → **Error 1033** | Коннектор не подключился. Смотрите логи `tunnel-named`: должны быть `Registered tunnel connection`. Уже включён `--protocol http2` (обходит проблемы QUIC/UDP). Подождите 30–60 c после старта. |
| Страница 404 «Tunnel» или «no such host» | Не сохранён Public Hostname (шаг 5) или зона не Active (шаг 1). |
| Ошибка авторизации токена в логах | Неверный/обрезанный `TUNNEL_TOKEN`. Скопируйте токен заново целиком. |
| Сайт грузится, но без стилей/данных | Frontend не запущен: `docker compose up -d frontend`. |
| Сертификат недоверенный | На named-туннеле TLS терминирует Cloudflare — недоверие быть не должно; почистите кэш браузера. |

---

## 11. Безопасность (обязательно перед «боевой» публикацией)
- По умолчанию стенд в `DEMO_MODE=true` → вход без пароля трактуется как **ADMIN**. Для реальной
  публикации в `.env`: `DEMO_MODE=false`, задайте сильный `JWT_SECRET` (≥32 симв.) и пароль БД,
  заведите реальных пользователей. Подробнее — `docs/SECURITY_ANALYSIS.md`.
- Можно закрыть доступ только для своих: Cloudflare **Zero Trust → Access → Applications** —
  добавить приложение на `asok.asokis.ai` с политикой по email (одноразовый код на почту).
- API-токен аккаунта Cloudflare проекту НЕ нужен — используется только токен туннеля.

**Вывод по B:** бизнес-видение в ТЗ заложено почти полностью. Два бизнес-разрыва **не покрыты
ни ТЗ, ни кодом**: (1) оценка **эффективности принятых мер** (стало ли лучше после меры);
(2) **проактивная защита от техсбоя** (риск-триггеры по метрикам). Плюс сквозной архитектурный
разрыв: **вся governance-петля в localStorage, а не в БД** — на разных устройствах роли видят
разные меры (демо-режим маскирует это одним браузером).
